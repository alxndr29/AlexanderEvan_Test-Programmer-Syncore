# Pertanyaan Jawaban
1 Mengapa setiap bahasa pemrograman itu terdapat framework, apa kegunaan framework ? Menurut saya terciptanya sebuah framework itu untuk mempermudah pekerjaan sehingga lebih rapih dan efisien dalam pengerjaannya. Apalagi jika project yang dikerjakan itu dalam sebuah team, tanpa framework akan beda-beda pattern design dari tiap programmer. Kalau pake framework sudah jelas harus create file di folder apa dsb. Juga kebanyakan framework juga sudah menerapkan konsep OOP sehingga bisa reusable juga dalam penggunakan function dan source.
2 Jelaskan mengenai fungsi dan penggunaan if else, while, while do, switch case, for, foreach? * if else itu digunakan untuk pembanding (menggunakan operator logika). if bisa berdiri sendiri namun else tidak. ada juga if, else if, dan else. * swith case tidak menggunakan operator logika. swith digunakan apabila 1 variabel punya banyak nilai sehingga lebih singkat dalam penulisannya. * for dan foreach adalah perulangan. Prinsipnya sama aja cuma kalau for itu bisa kita tentuin mulai darimana dan batasannya apa. * while dan do while. Kalau while itu dia melakukan perulangan kalau kondisinya memungkinkan, sedangkan do while itu akan melakukan perulangan terlebih dahulu kemudian memeriksa batasannya.
3 Apa itu constructor, method, class, object, variabel, parameter, function, procedure ? * constructor adalah sebuah function dalam class. function ini akan dipanggil ketika class tersebut dibuat. * method adalah potongan perintah, biasanya terdapat didalam class. * function mirip seperti method, cuma function itu lebih fleksible bisa diluar class dll. ada yang public maupun static. * class digunakan untuk merepresentasikan sebuah object yg didalamnya ada data member dan method. * object itu adalah class yang telah inisialisasikan. * parameter itu merupakan sebuah inputan yang akan dimasukan kedalam function. * procedure menurut saya adalah programming tanpa OOP dll. jadi biasanya cuma kumpulan banyak program dalam 1 file.
4 Perbedaan method=”Post” dan “Get” ? Sederhananya kalau di API itu POST untuk mengirimkan data dan GET untuk mengambil data. Kalau pake POST maka data yang dikirimkan itu tidak tampil di URL. Sedangkan kalau GET parameternya akan dituliskan ke URL. Sehingga lebih aman POST untuk melakukan update data karena orang tidak akan iseng mengirimkan parameter via URL.
5 Jelaskan konsep mengenai web service dan fungsi apa yang digunakan untuk memanggil web service client ? Web Service menurut saya sama saja seperti dengan API. Panggil web service bisa dengan curl atau guzzle. di front end bisa pake jquery atau axios. Konsepnya web service itu sebagai penyedia layanan saja, misal saat kita panggil web servicenya dengan url dan parameter yang telah ditentukan, maka dari web servicenya sendiri akan mengembalikan pesan atau data yang kita butuhkan.
6 Jelaskan mengenai MVC dan HMVC ! MVC = model, view, dan controller. Model itu merepresentasikan tabel yang ada di database, di model juga bisa kita buat custom function dan juga relasi. Controller itu sebagai penghubung antara view dan model. HMVC dan MVW sebenarnya tidak ada perbedaan berarti, namun kalau kerja secara team lebih baik menggunakan HMVC karena struktur foldernya lebih rapih sehingga kalau kita kerja pake git dsb sangat mengurangi resiko untuk conflict
7 Sebutkan fungi curl() dalam php! CURL itu sebenarnya extensi dari php. jadi CURL itu method yang digunakan untuk mengakses API dengan berbagai method. selain CURL yang biasa digunakan itu guzzle. kalau di javascript bisa pake ajax atau axios.
8 Bagaimana menampilkan sebuah element array dalam PHP ? Kalau hardcodenya bisa pake $element[index]. Bisa pake for loop, bisa pake foreach, atau segala macam perulangan yang penting ada nilai indexnya. Untuk untuk debug bisa pake var_dump atau print_r.
9 Perbedaan HTTP dan HTTPS ? bedanya cuma di S yang artinya "SECURE". Https lebih aman dan tidak mudah diretas. Tiap web ada protokolnya masing-masing.
10 Jelaskan Apa itu API? sebutkan contohnya? API itu adalah Application Programming Interface. Sederhananya itu api biasanya digunakan pada konsep microservices. Misal dalam 1 aplikasi ada banyak bahasa pemrograman, untuk komunikasinya itu lewat API. Biasanya yang di return itu dalam format json. API juga menggunakan HTTP method sprti get, post, put, delete. API juga ada authentication seperti JWT token dsb. Contohnya API itu kayak Tokopedia. Appsnya tokopedia sendiri itu banyak API didalam. Sebut aja api untuk menghitung ongkos kirim. Jadi tokopedia ngehit API perusahaan ekspedisi dengan parameter yang dibutuhkan, kemudian akan mengembalikan JSON yang akan ditampilkan ke UI pengguna.
